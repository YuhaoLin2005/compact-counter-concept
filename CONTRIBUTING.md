# 如何参与

这个仓库的核心是一份产品洞察和开放倡议。如果你认同"压缩次数 = 健康反向指标"这个方向，欢迎一起来推动。

## 贡献方式

### 技术实现

1. 为更多 LLM 工具补充监控脚本（Aider、Cursor、ChatGPT Web 插件等）
2. 将本概念转化为 PR 提交给官方项目（Claude Code、DeepSeek-TUI 等）
3. 完善本仓库的监控脚本和测试

### 产品与讨论

1. 在你的社区（V2EX、Reddit、Discord、Twitter）分享这个想法
2. 给你常用的 LLM 工具提交 Feature Request，附上本仓库的 ASCII 概念图
3. 提出改进意见——如果你觉得指标设计有缺陷，开 Issue 讨论

### 当前需要的帮助

- [ ] 实现一个实际可运行的压缩次数监控脚本（Python / Node / Rust）
- [ ] 为 DeepSeek-TUI 或 Claude Code 贡献状态栏补丁
- [ ] 为 ChatGPT Web 端编写浏览器插件
- [ ] 补充真实用户的压缩体验数据（你遇到过几次压缩后才意识到 AI 在退化？）

## 提交规范

- **讨论先行**：大幅改动前先开 Issue 讨论设计方向
- **Commit 格式**：`feat:` / `fix:` / `docs:` / `design:`（设计迭代用 `design:`）
- **语言**：中英文皆可，关键概念建议中英双语标注

---

> 这个仓库不是谁的附属品。它是一份独立的开放倡议——如果能让哪怕一个工具团队在状态栏加上压缩次数，这个仓库就完成了它的使命。
