# 核心理念：压缩次数 = 健康度反向指标

## 问题
AI 长对话中，用户无法感知模型何时开始丢失信息、产生幻觉。现有工具只显示上下文百分比（还剩多少空间），但用户更需要知道"已经丢了多少信息"。

## 解决方案
将每次上下文压缩事件计数并实时显示。压缩次数越多，模型失控风险越高。

## 创新点
- **从"预测"转向"记录"**：不预测何时压缩，只记录已发生的压缩。
- **从"仪表"转向"日志"**：累计次数比瞬时百分比更有决策价值。
- **普适性**：任何有上下文压缩机制的 Agent 都适用。

## 用户行动指南
- 0-1 次压缩：健康，可放心使用。
- 2-3 次压缩：警告，部分上下文已丢失，适合简单任务。
- 4+ 次压缩：严重，强烈建议 `/reset` 或 `/rewind`。
